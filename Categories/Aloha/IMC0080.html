Reprocessamento de vendas no conciliador



● Para realizar o reprocessamento das vendas no conciliador, vamos abrir o executar e digitar seguinte comando "%iberdir%\bin\grindq.exe/date 20240201" ajuste o campo data conforme a necessidade, mas sempre nesse padrão americano, ano, mês e dia.





● Ao executar o comando vai aparecer uma tela, dependendo do servidor, ela vai aparecer e sumir muito rapido, ou demorar um pouco.





● Caso após realizar esse processo, o conciliador não apresente todos os valores, podem fechar ele novamente, e vamos seguir o seguinte caminho no servidor da loja.

● C:\BootDrv\AlohaQS\20240204

● Reforço, alterem a data, para a data que está apresentando problemas.

● Ao acessar a pasta, verifiquem se o arquivo "TRANS.LOG" está com um tamanho acima de 600Kb, caso contrário acionem a NCR para verificação.



● Caso o arquivo TRANS.LOG esteja com um tamanho acima do informado acima, vamos seguir, o próximo arquivo a ser verificado se existe dentro da pasta é "GNDDBF30.XXX" caso esse arquivo não esteja na pasta, vocês podem entrar em outra pasta com a data mas antiga e copiar ele direto, sem problemas, ou se preferirem, podem baixar também conforme o link GNDDBF30.XXX.zip





● Feito isso, vamos executar o comando novamente "%iberdir%\bin\grindq.exe/date 20240204", na minha situação usei essa data, pois é referente ao meu problema, caso sua data seja diferente, altere conforme necessário.







● Se mesmo realizando todo processo, as vendas não tenham subido, acione a NCR com todas as evidências do processo realizado, para que eles possam analisar melhor.
