OCORREU UM ERRO NA REQUISIÇÃO - ALOHA





● O erro exibido acima ocorre devido à seguinte situação: como destacado na imagem, o terminal está configurado para operar em modo de redirecionamento. Assim, se o terminal para o qual ele foi redirecionado estiver desligado ou enfrentar algum problema que cause o travamento do SAT, essa mensagem será exibida, apontando para o terminal específico em questão. No exemplo apresentado, o terminal é o PHTERM5; no entanto, em seu caso, pode ser um terminal diferente. Portanto, é essencial prestar atenção ao nome do host (hostname) mencionado na mensagem, conforme ilustrado na imagem acima, onde o hostname do terminal é enfatizado.


● Já em acesso ao terminal que estava com problema (phterm5) conforme a imagem acima relata, vamos realizar testes no sat, você pode fazer conforme o processo Sat - Cupom fiscal


● Nota importante: Se o seu SAT estiver funcionando corretamente, será necessário adotar uma abordagem diferente. Isso indica que o problema não está no SAT propriamente dito, mas possivelmente em um erro de comunicação com o terminal indicado.





● No meu caso, realizei o processo do Sat - Cupom fiscal e mesmo assim o problema ainda persistiu.







● Nesta situação, adotaremos uma estratégia alternativa: acessaremos o NCR Fiscal Manager - BOH no servidor. A seguir, forneceremos todas as rotas possíveis para localizar esse software.









● Após abrir o NCR Fiscal Manager, vamos clicar em "Cadastros".





● Em seguida, vamos em "Máquinas".





● Na tela abaixo, vamos selecionar o terminal que está apresentando o problema, que no nosso caso é o PHTERM3, e clicamos em "Editar".





● Após clicar em "Editar", seremos redirecionados para tela abaixo, onde explico cada campo abaixo.

● Nome da Máquina: Hostname

● Trabalhar como Gateway: É o terminal para o qual ele vai ser redirecionado, e trabalhará com o sat desse terminal em específico.

● Redirecionar vendas para: Campo que selecionamos o terminal que vai compartilhar o sat para o terminal com problema.

● Vale ressaltar que, sempre valide se o próprio terminal que está redirecionado não possui sat próprio.

● Após realizar todas essas validações, e for constatado que realmente o terminal com problema não possui sat, ou o sat dele está com problema, vamos realizar o redirecionamento, nesse caso como o 5 está com problema, vamos ter que alterar para outro terminal com sat na loja.







● Bem, no meu caso, o terminal 2 tem sat próprio, então irei redirecionar as vendas desse terminal para ele.





● Após selecionar o terminal, clicamos em "Salvar" e fechamos a tela seguinte, conforme imagem abaixo.





● Em seguida, vamos clicar em "Utilitário".





● Clicamos em "Atualizar dados".





● Confirmamos a mensagem abaixo.





● Clique em OK após atualizar, para confirmar a atualização.





● Ressalto que dependendo da loja o processo pode demorar um pouco.


● No meu caso o processo deu certo.







● Caso o processo feito não dê certo, tire print de todos os sats da loja em funcionamento, e evidêncie para assim prosseguir com o ticket para NCR

● Caso seja uma loja fora de SP, acione a NCR para verificação.
