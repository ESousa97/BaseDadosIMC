PROCEDIMENTO PARA ATUALIZAÇÃO DE DLL NO ALOHA



Processo de DLL serve tanto para o erro 31, caso o pinpad esteja OK no software da Gertec, quanto para bandeiras de cartões não passando.


Siga o processo a risca para que dê certo.


Segue abaixo a tela de erro mais comum apresentado nos terminais com Aloha (KFC - PH)



Segue o link das DLL's DLL Aloha







1. Primeiro passo, vamos verificar o software da gertec, para testarmos o pinpad, e ver se está ok:





2. Ao fazer o processo conforme imagem acima, clique com o botão direito do mouse encima do icone da gertec, que irá abrir uma pequena caixa de diálogo onde clicamos em "show"





3. Assim que clicar em show, irá aparecer uma caixa de dialogo, nela iremos clicar encima do pinpad com o botão direito do mouse e em seguida "test" conforme imagem abaixo:





4. Após clicar em test, tem que aparecer a seguinte mensagem, se ficar dessa forma, significa que o pinpad está OK, e podemos continuar com o processo, caso dê errado, teremos que reinstalar o driver da gertec, conforme processo IMC0050


Obs.: Se por acaso mesmo aplicando o processo indicado acima, o problema no pinpad ainda persistir, pode orientar a loja a realizar a troca do pinpad.





5. Feito isso, vamos continuar, agora vamos realizar a atualização de DLL's no Aloha, copiamos as DLL's baixadas para o servidor da loja, podem jogar na pasta NCR dentro do disco local C, ou na TI-IMC, fica ao critério do analista





6. Agora vamos abrir o terminal da loja via VNC para pegar as DLL's que copiamos para o servidor, ao acessar abrimos o executar no terminal, e digitamos \\+hostname do servidor, ficará igual a imagem abaixo.


Obs.: Abaixo, também tera uma imagem mostrando como pegar o hostname do servidor, caso não saiba.
