Validando Ambiente Windows - Problema para abrir o Degust



É imperativo ressaltar a importância da validação para o ótimo desempenho do nosso sistema. Cada aspecto deste processo é fundamental para assegurar a estabilidade do sistema. Embora a execução do passo a passo esteja a cargo da Linx, cabe a nós a responsabilidade de realizar essa tarefa. Isso decorre do fato de que não é prerrogativa da equipe da Linx inspecionar o ambiente operacional, mas sim, tratar questões diretamente relacionadas ao sistema. Portanto, solicitamos a sua atenção e comprometimento para que nenhum detalhe seja negligenciado durante este processo.



● Segue abaixo alguns dos erros que podem ser corrigidos com o processo abaixo.



● Primeiro passo, acesse o terminal que está com problema para abrir o Degust.

➤ Ao acessar o terminal, vamos abrir o executar no terminal e digitar \\hostnamedoserver

➤ Faça a mesma coisa com o IP do servidor \\ipdoservidor.





● Após digitar, aperte o Enter, se a comunicação estiver OK, vocês terão uma tela parecida com essa.





● Agora vá até o servidor da loja, abra o executar nele e digite \\hostnamedoterminal

● Faça a mesma coisa com o IP do terminal \\ipdoterminal.





● Caso a comunicação esteja OK, vamos vão ter a seguinte tela.

● Caso contrário PADRONIZAÇÃO DE CONTA

● A conta deve está padronizada para proseguirmos com todo o processo abaixo.





● Bem, feito o passo a passo acima, vamos verificar outro detalhe, o armazenamento interno do terminal, abra o Meu Computador do Windows através do comando Windows + E do seu teclado, assim que abrir verifique se o disco C tem espaço suficiente.





● Outro detalhe para verificar é se o firewall do terminal com problema está ativo, abra o executar e digite firewall.cpl, a tela de vocês devem está da seguinte forma.





● Agora vamos verificar se o Windows está com os pacotes básicos para o funcionamento do Degust, abra o executar no terminal e digite dentro dele o seguinte comando control você será redirecionado instatâneamente para o painel de control, dentro dele coloque para exibir por Ícones pequenos, após vá até a opção Programa e Recursos.





● Dentro da opção Programas e Recursos, vamos até Ativar ou desativar recursos do Windows.





● Ao abrir a janela de Recursos do Windows vamos verificar algumas caixas de seleções.





● Segue abaixo a lista de opções que devem estar marcadas.

➤ .NET Framework 3.5 (inclui .NET 2.0 e 3.0) - E todos seus dependentes abaixo.

➤ .NET Framework 4.8 Advanced Services - E todos seus dependentes abaixo.

➤ Simple TCPIP services(i.e. echo, daytime etc)

➤ Suporte para Compartilhamento de Arquivos SMB 1.0/CIFS - E todos seus dependentes abaixo.







● Após ceritificar que todas as opções estão marcadas, podem apertar OK.

➤ A opção "Suporte para Compartilhamento de Arquivos SMB 1.0/CIFS" serve para comunicação \\ com terminais Windows 7.





● Vai aparecer a seguinte tela de carregamento para vocês, caso tenham marcado alguma opção.





● Após pesquisar localmente, caso ele não encontre a opção local, ele vai pedir para realizar o download das opções desejadas pelo Windows Update.





● Somente aguardar o download dos arquivos.





● Enquanto ele está baixando os arquivos, vamos voltar novamente no painel de controle através do comando control no executar, e ir até a opção Sistema.





● Ao entrar na tela abaixo, vamos até a opção Configurações avançadas do sistema.





● Na janela que abrir selecionamos a aba Avançado e clicamos em Configurações....





● Na aba Efeitos Visuais selecionamos a opção Ajustar para obter um melhor desempenho.





● Agora clicamos na aba Avançado dentro dela vamos até a opção Alterar





● Na janela seguinte, desmarcamos a opção Gerenciar automaticamente o tamnho do arquivo de paginação de todas as unidades, selecionamos Sem arquivo de paginação e clicamos em Definir.





● Se aparecer uma caixa de diálogo para confirmação, apenas clicar em Sim.





● Após isso na janela anterior marcamos a opção Serviços em segundo plano e aplicamos a configuração.





● Clicamos em reiniciar depois, pois no caso apresentado estou baixando atualizações ainda pelo Windows Update.





● Agora vamos até o disco local C ou Meu computador, vamos clicar com o botão direito do mouse encima do disco C, e ir até Propriedades, nas propriedades do disco C, vamos até a aba Hardware, e propriedades novamente.





● Nas propriedades vamos até a opção Alterar Configurações.





● Vamos até a aba Políticas.





● Desmarcamos a opção Habilitar gravação em cache no dispositivo, e clicamos em OK.





● Após realizar a alteração, vamos abrir o executar novamente no terminal e digitar ncpa.cpl, assim abrimos direto a placa de rede do terminal.





● Vamos clicar com o botão direito do mouse encima de Ethernet e vamos até Propriedades.





● Dentro das propriedades da placa de rede, vamos certificar que a opção Protocolo IP Versão 6(TCP/IPv6) esteja desmarcada (Em caso de lojas que ficam em aeroportos essa opção deve está marcada.), selecionamos a opção Protocolo IP Versão 4(TCP/IPv4) e vamos até propriedades.





● Validem se o terminal está com a DNS correta.

● Servidor DNS preferencial 8.8.8.8

● Servidor DNS alterantivo 8.8.4.4

● Após certificarem ou corrigirem, vamos na opção Avançado...





● Vamos até a aba WINS.





● Na aba WINS vamo marcar a seguinte opção Ativar NetBios sobre TCP/IP e apertar OK.





● Lembrando que os recursos do Windows ainda estão sendo aplicados.





● Abra a bandeja de ícones na barra de tarefas do Windows, vocês iram ver em alguns terminais esse quadrado azul, que é o ícone da Intel, clicamos com o botão direito do mouse sobre dele, e vamos até a opção Propriedades gráficas....





● Vai começar a carrega o painel da Intel.





● Vamos na opção 3D.





● Marcamos a opção Desempenho e clicamos em Aplicar, após podem fechar.





● Caso peça para confirmar, cliquem em Sim.





● Assim que finalizar a instalação dos recursos do Windows, clicamos em Reiniciar agora.





● Nessa etapa somente aguardar o Windows instalar todas as mudanças que fizemos no sistema e testar novamente quando voltar.





● Caso tenha que instalar o sistema novamente, consulte INSTALAÇÃO DE DEGUST
