Não é possível localizar o servidor de arquivos



● Acesse todos os terminais da loja e identifique o terminal que tem o botão Tornar Servidor de Arquivos.





● Assim que identificar o terminal com esse botão.





● Vamos abrir o Executar no servidor, e digitar \\hostname do terminal.





● Assim que acessar, vamos no seguinte caminho BootDrv\AlohaQS\ vamos procurar o arquivo com o seguinte nome Downtime.ini.





● Ao abrir esse arquivo com o bloco de notas, verifique se ele possui a numeração do terminal conforme bate no hostname, no meu caso é terminal 2.





● Caso esteja correto, vamos no mesmo caminho, só que no servidor C:\BootDrv\AlohaQS\ dentro da pasta vamos procurar o mesmo arquivo Downtime.ini.





● Abrimos o arquivo com o bloco de notas, e validamos se está OK, caso não, iremos verificar mas uma coisa.





● Antes de acionar a NCR, vamos verificar os serviços do servidor, caso não saiba onde fica, abra o executar e digite services.msc, nele vamos verificar se o serviço CtlSvr está em execução, caso não pode iniciar.









● Se o serviço iniciou conforme o esperado, vamos aguardar uns 3 min para verificar se todos os terminais vão subir.





● Conforme disse, devemos esperar 3 min ou mas um pouco, pois as vezes é só questão de tempo para o terminal voltar.





● Caso dê algum problema, acione de imediato a NCR. NCR.
