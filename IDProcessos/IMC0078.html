Aprimorando Ambiente de Trabalho: Checagem de Padronização do PDV - Terminal



● A primeira medida para garantir a padronização do terminal é realizar o acesso direto ao terminal que requer inspeção.





● Ao acessar o terminal, a primeira etapa consiste em validar se a conta local está alinhada com os padrões do servidor da loja.





● O próximo passo envolve abrir o Prompt de Comando (CMD) sem privilégios de administrador para verificar se a conta local está configurada corretamente ou se apresenta alguma modificação.





● Em seguida, abra o no terminal e digite `\\` seguido pelo nome do host do servidor, sem espaços, conforme ilustrado na imagem abaixo.





● Se a conexão for estabelecida com sucesso, você terá acesso a todas as pastas disponíveis e compartilhadas no servidor.





● Realizamos o mesmo procedimento no servidor: abrimos o Executar pressionando "Windows + R", mas desta vez digitaremos "\\"" seguido pelo nome do host do terminal.





● Se o procedimento for bem-sucedido, vocês terão acesso normal às pastas que foram compartilhadas a partir do terminal, conforme ilustrado na imagem.





● O próximo passo é verificar se o firewall do terminal está ativado. Para isso, abra o Executar no terminal e digite "firewall.cpl".





● Após verificar o firewall, abra novamente o Executar e digite "ncpa.cpl" para acessar as configurações de rede.





● Ao acessar as configurações da placa de rede, clique com o botão direito do mouse sobre a conexão desejada e selecione "Propriedades".





● Na janela que se abre, localize e selecione a opção "Protocolo IP Versão 4 (TCP/IPv4)", e então clique em "Propriedades" para ajustar as configurações de IP.





● Verifique se o terminal está configurado com o DNS padrão sendo 8.8.8.8 e 8.8.4.4. Se estiver correto, clique em "Avançado...". Caso contrário, ajuste essas configurações para os valores mencionados antes de prosseguir.





● Na janela "Configurações TCP/IP avançadas", desmarque a opção "Métrica automática", se estiver marcada, e defina o valor para "1". Em seguida, clique na guia "WINS", seguindo a indicação da seta vermelha.





● Se a opção "Configuração NetBIOS" estiver configurada como "Padrão", altere para "Ativar NetBIOS sobre TCP/IP", conforme ilustrado na imagem abaixo.





● Após realizar todas essas alterações, prossiga clicando em "OK" para fechar cada janela aberta e efetivar as configurações ajustadas.


● Abra novamente o Executar e digite "control system". Na tela que se abrirá, procure pela opção "Configurações avançadas do sistema".


● Agora, clique na aba "Nome do Computador" ("Computer Name") para validar se o hostname está conforme o padrão estabelecido pela loja, e também verifique se o "Grupo de Trabalho" ("Workgroup") está configurado de acordo com o servidor.


● Observação: Caso o hostname ou o "Grupo de Trabalho" não estejam configurados corretamente, clique em "Alterar..." ("Change...") para ajustá-los conforme necessário.





● Para validar o grupo de trabalho e a nomenclatura do hostname do terminal, abra o Executar no servidor pressionando `Windows + R` e digite "control system". Na tela que se abrir, vá até a opção "Configurações avançadas do sistema" e proceda com a validação conforme indicado na imagem abaixo.





● Se você tiver alguma dúvida ou se o seu terminal estiver completamente fora do padrão, retome o primeiro processo para padronizar a conta da máquina de acordo com os requisitos específicos da loja. PADRONIZAÇÃO DE CONTA .



● Para otimizar a performance do terminal ajustando as opções de desempenho, siga as instruções a seguir: PERFORMANCE DO WINDOWS .



● As instruções fornecidas anteriormente são aplicáveis também a terminais que operam com o sistema Aloha, visto que se referem a ajustes de contas e otimização de desempenho no Windows.
